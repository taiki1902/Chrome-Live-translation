# Helium Live Translator

Helium / Chromium の現在のタブ音声を取り込み、OpenAIで文字起こし・翻訳して、ページ上へ字幕表示する Manifest V3 拡張機能です。

> **現在は BYOK（Bring Your Own Key）方式です。** 利用者自身の OpenAI API キーが必要で、API 利用料金が発生する場合があります。

## 主な機能

- 配信・動画・通話サイトなど、選択したタブの音声を取得
- 3〜10秒単位の準リアルタイム文字起こしと翻訳
- 区間を重ねて録音し、境界で欠けやすい単語を補完
- 重複字幕の自動除去
- 無音区間のローカル判定によるAPI利用量の削減
- 翻訳字幕と原文字幕の同時表示
- 画面上/下、文字サイズ、固有名詞辞書の設定
- Shadow DOMによるサイトCSSとの衝突回避
- `Alt + Shift + L` で開始・停止
- APIキー、音声、字幕を開発者サーバーへ送信しない構成

## 対応環境

- Helium Browser
- Google Chrome / Chromium 系ブラウザ 116以降
- 通常の `http://` / `https://` ページ

Helium は Chromium 拡張機能をサポートしています。`chrome://`、ブラウザ設定ページ、Chrome Web Store、ローカルファイルなどでは動作しません。一部のDRM保護コンテンツや、動画要素そのものを直接フルスクリーン化するサイトでは、取得・字幕重ね表示が制限される場合があります。

## インストール

### 開発版を読み込む

1. Releases の ZIP を展開するか、このリポジトリをクローンします。
2. Helium の拡張機能管理ページを開きます。
3. **デベロッパーモード**を有効にします。
4. **パッケージ化されていない拡張機能を読み込む**から、`manifest.json` があるフォルダーを選びます。

### ソースから配布ZIPを作る

```bash
npm install
npm run check
npm run build
```

`dist/helium-live-translator-v0.1.0.zip` が生成されます。

## 使い方

1. OpenAI Platform でAPIキーを発行します。
2. 拡張機能のポップアップへキーを入力します。
3. 音声言語と字幕言語を選びます。
4. 配信・動画のタブで **翻訳を開始** を押します。
5. 終了時は **翻訳を停止** を押します。

初期設定は、文字起こしに `gpt-4o-mini-transcribe`、翻訳に `gpt-5-mini` を使用します。利用可能モデルはOpenAIアカウントによって異なる場合があるため、詳細設定から変更できます。

## 遅延と料金

これは音声ストリーミング専用プロトコルではなく、短い独立音声ファイルを順番に処理する方式です。通常は設定した音声区間にAPI処理時間を加えた遅延が発生します。初期値4秒の場合、回線とAPI混雑状況により、おおむね数秒以上遅れて表示されます。

音声区間を短くすると字幕更新は速くなりますが、APIリクエスト数が増えます。無音スキップを有効にし、用途に応じて4〜6秒へ設定することを推奨します。

## 権限

| 権限                       | 用途                                                   |
| -------------------------- | ------------------------------------------------------ |
| `activeTab`                | 利用者が選択した現在のタブだけを操作                   |
| `tabCapture`               | 選択タブの音声ストリームIDを取得                       |
| `offscreen`                | バックグラウンドでMediaStream / MediaRecorderを使用    |
| `scripting`                | インストール前から開いていたタブへ字幕スクリプトを注入 |
| `storage`                  | APIキーと表示設定をブラウザ内へ保存                    |
| `https://api.openai.com/*` | OpenAI APIへ文字起こし・翻訳を直接送信                 |

## プライバシーとセキュリティ

- 分析・広告・テレメトリはありません。
- 開発者管理のバックエンドはありません。
- APIキーは `chrome.storage.local` に保存されます。ただし、これはOSの秘密情報保管庫ではありません。共有PCや侵害されたブラウザプロファイルでは読み取られる可能性があります。
- 音声は短い区間に分けられ、OpenAI文字起こしAPIへ直接送信されます。
- 文字起こし結果は翻訳のためOpenAI Responses APIへ送信され、リクエストでは `store: false` を指定します。
- 詳細は [PRIVACY.md](PRIVACY.md) と [SECURITY.md](SECURITY.md) を参照してください。

## 既知の制約

- 同時に翻訳できるタブは1つです。
- API処理速度がライブ音声に追いつかない場合、メモリ増加を防ぐため古い未処理区間を破棄します。
- 楽曲、複数人同時発話、効果音が大きい配信では認識精度が低下します。
- 字幕は音声と完全同期ではありません。
- 翻訳品質・利用可能モデル・料金は外部API仕様に依存します。
